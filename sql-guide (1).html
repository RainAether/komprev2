01

INSERT

DML

INSERT digunakan untuk menambah data baru ke dalam tabel. Bayangkan kayak nulis data baru ke dalam buku catatan.

SQL Query
-- Tambah data produk
INSERT INTO products (id, name, category, price, stock)
VALUES
  (1, 'Laptop ASUS',  'Elektronik', 8500000, 10),
  (2, 'Mouse Logitech','Elektronik', 350000,  50),
  (3, 'Meja Kayu',     'Furnitur',  1200000, 5),
  (4, 'Kursi Ergonomis','Furnitur',  2300000, 8);

-- Tambah data pelanggan
INSERT INTO customers (id, name, city, email)
VALUES
  (1, 'Budi Santoso', 'Jakarta', 'budi@email.com'),
  (2, 'Siti Rahayu',  'Surabaya','siti@email.com'),
  (3, 'Andi Wijaya',  'Bandung', 'andi@email.com');

-- Tambah data pesanan
INSERT INTO orders (id, customer_id, product_id, qty, total)
VALUES
  (1, 1, 1, 1, 8500000),
  (2, 2, 2, 2, 700000),
  (3, 3, 3, 1, 1200000),
  (4, 1, 4, 1, 2300000);

02

UPDATE

DML

UPDATE digunakan untuk mengubah data yang sudah ada. Jangan lupa pakai WHERE, kalau tidak semua baris akan ikut berubah!

SQL Query
-- Naikkan harga Laptop ASUS jadi 9.000.000
UPDATE products
SET    price = 9000000, stock = 15
WHERE  id = 1;
Output — Setelah UPDATE
idnameprice (sebelum)price (sesudah)stock
1Laptop ASUS8,500,0009,000,00015
Hanya baris dengan id = 1 yang berubah. 1 row affected.
03

ALTER TABLE

DDL

ALTER TABLE digunakan untuk mengubah struktur tabel — menambah kolom baru, menghapus kolom, atau mengubah tipe data. Kayak renovasi rumah!

SQL Query
-- Tambah kolom 'discount' ke tabel products
ALTER TABLE products
ADD COLUMN discount DECIMAL(5,2) DEFAULT 0;

-- Ubah tipe kolom 'stock' menjadi BIGINT
ALTER TABLE products
MODIFY COLUMN stock BIGINT;
Output — Struktur tabel products setelah ALTER

Kolom discount berhasil ditambahkan. Semua baris lama otomatis mendapat nilai default = 0.
04

ORDER BY

SELECT

ORDER BY mengurutkan hasil query. Gunakan ASC (kecil ke besar) atau DESC (besar ke kecil).

SQL Query
-- Tampilkan produk dari harga termahal ke termurah
SELECT name, category, price
FROM   products
ORDER BY price DESC;
Output — 4 rows returned
#namecategoryprice
1Laptop ASUSElektronik9,000,000
2Kursi ErgonomisFurnitur2,300,000
3Meja KayuFurnitur1,200,000
4Mouse LogitechElektronik350,000
Data terurut dari harga 9.000.000 (paling mahal) ke 350.000 (paling murah).
05

GROUP BY

SELECT

GROUP BY mengelompokkan data berdasarkan kolom tertentu. Biasanya dikombinasikan dengan fungsi agregat seperti COUNT, SUM, AVG.

SQL Query
-- Hitung jumlah produk & rata-rata harga per kategori
SELECT
  category,
  COUNT(*)        AS jumlah_produk,
  AVG(price)      AS rata_harga,
  SUM(stock)      AS total_stok
FROM   products
GROUP BY category;
Output — 2 rows returned (2 kategori)
categoryjumlah_produkrata_hargatotal_stok
Elektronik24,675,00065
Furnitur21,750,00013
Data 4 produk berhasil diringkas menjadi 2 baris berdasarkan kategorinya. Rata-rata harga Elektronik = (9.000.000 + 350.000) ÷ 2.
06

INNER JOIN

SELECT

INNER JOIN menggabungkan dua tabel dan hanya menampilkan baris yang ada di kedua tabel. Kalau salah satu tidak cocok, tidak akan muncul.

SQL Query
-- Tampilkan pesanan beserta nama pelanggan & nama produk
SELECT
  o.id         AS order_id,
  c.name       AS pelanggan,
  p.name       AS produk,
  o.qty,
  o.total
FROM        orders    AS o
INNER JOIN  customers AS c  ON o.customer_id = c.id
INNER JOIN  products  AS p  ON o.product_id  = p.id;
Output — 4 rows returned
order_idpelangganprodukqtytotal
1Budi SantosoLaptop ASUS19,000,000
2Siti RahayuMouse Logitech2700,000
3Andi WijayaMeja Kayu11,200,000
4Budi SantosoKursi Ergonomis12,300,000
3 tabel digabung sekaligus! Data dari orders, customers, dan products tersambung lewat kolom ID yang cocok.
07

LEFT JOIN

SELECT

LEFT JOIN menampilkan semua data dari tabel kiri, meskipun tidak ada pasangan di tabel kanan. Yang tidak punya pasangan akan tampil sebagai NULL.

SQL Query
-- Tambah pelanggan baru yang belum pernah pesan
INSERT INTO customers VALUES (4, 'Dewi Kusuma', 'Medan', 'dewi@email.com');

-- LEFT JOIN: tampilkan semua pelanggan + pesanan mereka
SELECT
  c.name    AS pelanggan,
  c.city,
  o.id      AS order_id,
  o.total
FROM      customers AS c
LEFT JOIN orders    AS o  ON c.id = o.customer_id;
Output — 5 rows returned (Dewi muncul walau belum pesan!)
pelanggancityorder_idtotal
Budi SantosoJakarta19,000,000
Budi SantosoJakarta42,300,000
Siti RahayuSurabaya2700,000
Andi WijayaBandung31,200,000
Dewi Kusuma ★MedanNULLNULL
Dewi Kusuma tetap muncul meskipun belum pernah memesan, dengan nilai NULL di kolom order. Inilah perbedaan utama LEFT JOIN vs INNER JOIN.
08

VIEW

SPECIAL

VIEW adalah "tabel virtual" yang menyimpan sebuah query. Kita bisa memanggilnya kapan saja seperti tabel biasa — tanpa harus nulis query panjang berulang-ulang.

SQL Query
-- Buat VIEW ringkasan pesanan
CREATE VIEW v_ringkasan_pesanan AS
  SELECT
    c.name       AS pelanggan,
    c.city,
    COUNT(o.id)  AS total_order,
    SUM(o.total) AS total_belanja
  FROM      customers c
  LEFT JOIN orders    o  ON c.id = o.customer_id
  GROUP BY  c.id, c.name, c.city;

-- Gunakan VIEW semudah SELECT dari tabel biasa
SELECT * FROM v_ringkasan_pesanan
ORDER BY total_belanja DESC;
Output — v_ringkasan_pesanan
pelanggancitytotal_ordertotal_belanja
Budi SantosoJakarta211,300,000
Andi WijayaBandung11,200,000
Siti RahayuSurabaya1700,000
Dewi KusumaMedan0NULL
View v_ringkasan_pesanan menyimpan query kompleks. Kita tinggal panggil SELECT * FROM v_ringkasan_pesanan kapanpun — simpel!
09

TRIGGER

SPECIAL

TRIGGER adalah aksi otomatis yang berjalan ketika ada INSERT, UPDATE, atau DELETE. Bayangkan seperti alarm — kalau ada kejadian tertentu, trigger langsung bereaksi sendiri.

SQL Query
-- Buat tabel log untuk mencatat perubahan stok
CREATE TABLE log_stok (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  product_id  INT,
  stok_lama   INT,
  stok_baru   INT,
  waktu       DATETIME DEFAULT NOW()
);

-- Buat TRIGGER: otomatis catat log saat stok diupdate
CREATE TRIGGER trg_update_stok
AFTER UPDATE ON products
FOR EACH ROW
BEGIN
  IF OLD.stock != NEW.stock THEN
    INSERT INTO log_stok (product_id, stok_lama, stok_baru)
    VALUES (NEW.id, OLD.stock, NEW.stock);
  END IF;
END;

-- Simulasi: update stok Mouse Logitech
UPDATE products
SET    stock = 45
WHERE  id = 2;

-- Lihat log yang otomatis tercatat
SELECT * FROM log_stok;
Output — log_stok (terisi otomatis tanpa ditulis manual!)
idproduct_idstok_lamastok_baruwaktu
1250452024-01-15 10:32:17
Saat UPDATE products SET stock = 45 dijalankan, trigger langsung otomatis menyimpan log perubahan ke tabel log_stok — tanpa kita tulis INSERT-nya!
